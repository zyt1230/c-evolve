- makefile 文件

```makefile
# 第一种方法
s.i:s.c
        g++ -E s.c -o s.i
s.s:s.i 
        g++ -S s.i -o s.s
s.o:s.s
        g++ -c s.s -o s.o
s:s.o
        g++ s.o -o s
clean:
        rm -f s.i s.o s.s


# 另一种写法
# 这里有个all: 会指定默认目标为 all的s，就不会是第一条了
all: s  # 默认目标

s: s.o
        g++ s.o -o s

s.o: s.s
        g++ -c s.s -o s.o

s.s: s.i
        g++ -S s.i -o s.s

s.i: s.c
        g++ -E s.c -o s.i

clean:  # 没有依赖，直接执行目标
        rm -f s.i s.s s.o s
```

- 如何执行

```makefile
这里是以 第一种makefile写法为主
make                       #执行 Makefile 的第一个目标（默认 s.i，只预处理）

make s                       #明确构建可执行文件 s（完整流程）

make s.i                     #只进行预处理

make clean                   #删除生成的文件


```

## 3.3、语法格式

```makefile
目标：目标依赖
 命令

注意事项：
 命令必须使用tab键开头，一般是shell指令
```

## 3.4、目标

- 默认目标

一个Makefile里面可以有多个目标，一般会选择第一个当做默认目标也就是make默认执行的目标 。

- 多目标

一个规则中有多个目标，多个目标具有相同的生成命令。

```makefile
clean distclean: # 2个目标 ，执行make disclean和make clean是一样的
    rm hello.[^cpp] hello
```

- 多规则目标

```makefile
多个规则可以是同一目标
使用make会执行下面所有的目标
all:test1
all:test2
test1:
    @echo "hello"
test2:
    @echo "world"
```

- 伪目标

看clean位置的注释

```makefile
并不是一个真正的文件名，可以看做是一个标签。
无依赖，相比一般文件，不会重新生成、执行。
伪目标：可以无条件执行，相当于对应的指令。
例如：
.PHONY: clean all  # 声明 clean 和 all 是伪目标

all: program

program: main.o utils.o
    g++ main.o utils.o -o program

main.o: main.cpp
    g++ -c main.cpp -o main.o

utils.o: utils.cpp
    g++ -c utils.cpp -o utils.o

clean:
    rm -f *.o program   #否则，如果目录中意外出现一个名为 clean 的文件，make clean 将不会执行删除操作！
```

## 3.5、目标依赖

- 文件时间戳

**文件时间戳（File Timestamps）** 是 `make` 工具用来决定**是否需要重新构建目标（target）** 的关键机制。`make` 通过比较 **目标文件** 和 **依赖文件** 的最后修改时间（`last modification time`）来判断是否需要执行规则中的命令。

工作流程：

1. **检查目标文件是否存在**
   
   * 如果目标文件 **不存在** → **执行规则中的命令**（构建目标）。
   
   * 如果目标文件 **存在** → 进入下一步。

2. **比较目标文件和依赖文件的时间戳**
   
   * 如果 **依赖文件比目标文件新**（`dependency is newer than target`）→ **执行规则中的命令**（重新构建目标）。
   
   * 如果 **目标文件是最新的** → **不执行任何操作**（`make: 'target' is up to date`）。

```makefile
stat 目标 #查看时间戳
--------
touch program   # 让 program 比依赖文件 "更旧"，触发重新编译
make
--------
program:main.c FORCE
    gcc main.c -o program

FORCE:  # 伪目标，强制重新编译
.PHONY: FORCE
```

## 5、变量

### 5.1、变量基础

- 变量命名：变量 = 变量值

- 变量的赋值：

=   直接赋值

+=  追加赋值

?=  条件赋值----如果之前没有值，则变量赋值，如果之前有值，则不就行变量赋值

- 变量的使用  `$(变量名) 或 ${变量名}`

```cpp
var1 = main.o  #创建变量
var1 += operate.o  #变量追加
all:operate
operate:${var1}  #使用变量
    #打印var1
  echo ${var1}
    g++ $^ -o $@
%.o:%.cpp
    g++ -c $^ -o $@
clean:
    rm -f *.o ope
```

### 5.3、变量类型

```makefile
1）立即展开变量
 使用:=操作符进行赋值
 在解析阶段直接赋值常量字符串
2）延迟展开变量
 使用=操作符进行赋值
 将最后一次赋值的结果给变量名使用
3）注意事项
 一般在目标、目标依赖中使用立即展开赋值
 在命令中一般使用延迟展开赋值变量
```

### 5.4、变量的外部传递

```makefile
make ARCH=g++
```

- 例子

```makefile
var1 = main.o          #定义一个变量并赋值
var1 += operator.o #给变量追加值
CC:=g++
#第一个依赖关系
all:operator
operator:$(var1)

 $(CC) $^ -o $@

%.o:%.cpp
 $(CC) -c $< -o $@
clean:
 rm -f *.o operator         
```

### 5.5、条件执行

```makefile
ifeq 、else 、endif

```

```cpp

 条件语句从ifeq开始执行，括号与关键字自减使用空格隔开
 括号里面挨着括号处，不允许加空格
```
